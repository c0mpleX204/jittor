# AutoDL 本地固定测试集实验指令

下面命令默认你已经进入云算力平台的 `starter_code` 目录。

```bash
cd /root/code2/release4/starter_code
```

如果环境还没装依赖，先执行：

```bash
python -m pip install -r requirements.txt
```

## 0. 把实验输出切到数据盘

AutoDL 系统盘较小，建议把权重、测试集、预测结果、打分结果都放到数据盘：

```bash
bash tools/autodl_prepare_datadisk.sh /root/autodl-tmp/jittor_pcd
```

这个脚本会把下面几个目录移动到 `/root/autodl-tmp/jittor_pcd`，并在 `starter_code` 下建立软链接：

```text
experiments
result
true_test
noisy_test
denoisy_test
denoisy_test_raw
```

检查空间：

```bash
df -h
du -sh experiments result true_test noisy_test denoisy_test denoisy_test_raw 2>/dev/null
```

如果系统盘仍然很满，先定位 `/root/code2` 里的大目录和大文件：

```bash
du -h --max-depth=2 /root/code2 | sort -h | tail -50
find /root/code2 -type f \( -name "*.tar.gz" -o -name "*.zip" -o -name "*.pkl" -o -name "*.npy" \) -printf "%s %p\n" | sort -n | tail -50
```

确认是不再需要的压缩包后，可以移动到数据盘归档：

```bash
mkdir -p /root/autodl-tmp/jittor_pcd/archives
mv /root/code2/*.tar.gz /root/code2/*.zip /root/autodl-tmp/jittor_pcd/archives/ 2>/dev/null || true
```

## 1. 生成本地干净测试集 true_test

从 `dataset_train` 的 OBJ 网格里抽样，生成固定的 `true_test/shapenet/.../clean.npy`。
由于 `true_test` 已经是软链接，实际数据会写到 `/root/autodl-tmp/jittor_pcd/true_test`。

```bash
python tools/local_make_true_test.py \
  --dataset-dir dataset_train \
  --datalist datalist/train.txt \
  --out-dir true_test \
  --list-out datalist/true_test.txt \
  --num-models 200 \
  --points 50000 \
  --seed 2026 \
  --overwrite
```

说明：

- `--num-models 200` 可以改成 50、100、500。
- `--points 50000` 对齐赛题测试点云规模。
- `--overwrite` 会删除旧的 `true_test` 后重新生成。

## 2. 给 true_test 加噪，生成 noisy_test

```bash
python tools/local_make_noisy_test.py \
  --gt-dir true_test \
  --gt-list datalist/true_test.txt \
  --out-dir noisy_test \
  --list-out datalist/noisy_test.txt \
  --noise-type laplace \
  --std-min 0.005 \
  --std-max 0.020 \
  --seed 2026 \
  --overwrite
```

这一步会生成：

```text
noisy_test/shapenet/.../noisy.npy
noisy_test/noise_meta.csv
noisy_test/noise_meta.json
datalist/noisy_test.txt
```

## 3. 训练 baseline，权重写入数据盘

使用数据盘训练配置：

```bash
mkdir -p result/logs

python -u run.py --task configs/task/train_vm_autodl_tmp.yaml 2>&1 | tee result/logs/train_baseline.log
```

权重会保存到：

```text
/root/autodl-tmp/jittor_pcd/experiments/vm/checkpoint_0.pkl
/root/autodl-tmp/jittor_pcd/experiments/vm/checkpoint_1.pkl
...
/root/autodl-tmp/jittor_pcd/experiments/vm/checkpoint_99.pkl
```

## 4. 对某个 checkpoint 推理 noisy_test

比如测试第 99 个 epoch：

```bash
sed -i "s#^load_ckpt:.*#load_ckpt: /root/autodl-tmp/jittor_pcd/experiments/vm/checkpoint_99.pkl#" configs/task/predict_noisy_test_autodl_tmp.yaml
```

删除旧预测并重新推理：

```bash
rm -rf /root/autodl-tmp/jittor_pcd/denoisy_test_raw /root/autodl-tmp/jittor_pcd/denoisy_test

python -u run.py --task configs/task/predict_noisy_test_autodl_tmp.yaml 2>&1 | tee result/logs/predict_epoch99.log
```

整理 baseline writer 的临时输出，生成干净的 `denoisy_test/shapenet/.../denoised.npy`：

```bash
python tools/local_collect_predictions.py \
  --raw-dir /root/autodl-tmp/jittor_pcd/denoisy_test_raw \
  --out-dir /root/autodl-tmp/jittor_pcd/denoisy_test \
  --list datalist/noisy_test.txt \
  --overwrite
```

## 5. 打分并生成可视化文件

```bash
python tools/local_score_and_visualize.py \
  --gt-dir /root/autodl-tmp/jittor_pcd/true_test \
  --noisy-dir /root/autodl-tmp/jittor_pcd/noisy_test \
  --pred-dir /root/autodl-tmp/jittor_pcd/denoisy_test \
  --mesh-dir dataset_train \
  --result-dir /root/autodl-tmp/jittor_pcd/result/epoch_99 \
  --workers 8 \
  --visualize-count 8 \
  --overwrite
```

输出位置：

```text
/root/autodl-tmp/jittor_pcd/result/epoch_99/scores/summary_cd_p2s_scores.json
/root/autodl-tmp/jittor_pcd/result/epoch_99/scores/per_sample_cd_p2s_scores.csv
/root/autodl-tmp/jittor_pcd/result/epoch_99/visualization/
```

可视化文件是 `.ply`，可以用 CloudCompare 或 MeshLab 打开。

## 6. 一次测试多个 checkpoint

例如只测 20、40、60、80、99：

```bash
mkdir -p result/logs

for E in 20 40 60 80 99; do
  sed -i "s#^load_ckpt:.*#load_ckpt: /root/autodl-tmp/jittor_pcd/experiments/vm/checkpoint_${E}.pkl#" configs/task/predict_noisy_test_autodl_tmp.yaml

  rm -rf /root/autodl-tmp/jittor_pcd/denoisy_test_raw /root/autodl-tmp/jittor_pcd/denoisy_test

  python -u run.py --task configs/task/predict_noisy_test_autodl_tmp.yaml 2>&1 | tee result/logs/predict_epoch${E}.log

  python tools/local_collect_predictions.py \
    --raw-dir /root/autodl-tmp/jittor_pcd/denoisy_test_raw \
    --out-dir /root/autodl-tmp/jittor_pcd/denoisy_test \
    --list datalist/noisy_test.txt \
    --overwrite

  python tools/local_score_and_visualize.py \
    --gt-dir /root/autodl-tmp/jittor_pcd/true_test \
    --noisy-dir /root/autodl-tmp/jittor_pcd/noisy_test \
    --pred-dir /root/autodl-tmp/jittor_pcd/denoisy_test \
    --mesh-dir dataset_train \
    --result-dir /root/autodl-tmp/jittor_pcd/result/epoch_${E} \
    --workers 8 \
    --visualize-count 8 \
    --overwrite
done
```

每次实验结束后，把 `/root/autodl-tmp/jittor_pcd/result` 文件夹下载回 Windows 归档即可。

## 7. 如果训练已经开始且权重在系统盘

如果当前训练还在跑，先不要直接删除目录。可以先把已经保存的权重移动到数据盘并建立软链接：

```bash
mkdir -p /root/autodl-tmp/jittor_pcd/experiments

if [ -d experiments/vm ] && [ ! -L experiments/vm ]; then
  mkdir -p /root/autodl-tmp/jittor_pcd/experiments/vm
  mv experiments/vm/* /root/autodl-tmp/jittor_pcd/experiments/vm/ 2>/dev/null || true
  rmdir experiments/vm 2>/dev/null || true
  ln -s /root/autodl-tmp/jittor_pcd/experiments/vm experiments/vm
fi
```

之后新保存的 checkpoint 会落到数据盘对应目录。

如果系统盘已经很紧张，建议等当前 epoch 保存完后按 `Ctrl+C` 停一下，再执行上面的迁移命令，然后用 `configs/task/train_vm_autodl_tmp.yaml` 重新开始训练。

## 8. 用最佳 checkpoint 推理官方测试集并打包提交

本地 `true_test/noisy_test/denoisy_test` 只用于选 checkpoint 和做实验记录。

正式参赛提交时，要对官方：

```text
dataset_test_noisy
```

进行推理。假设你本地评测后决定使用 `checkpoint_60.pkl`：

```bash
cd /root/code2/release4/starter_code

sed -i "s#^load_ckpt:.*#load_ckpt: /root/autodl-tmp/jittor_pcd/experiments/vm/checkpoint_60.pkl#" configs/task/predict_submit_autodl_tmp.yaml

rm -rf /root/autodl-tmp/jittor_pcd/submit_result_raw /root/autodl-tmp/jittor_pcd/submit_result

python -u run.py --task configs/task/predict_submit_autodl_tmp.yaml 2>&1 | tee /root/autodl-tmp/jittor_pcd/result/logs/predict_submit_epoch60.log
```

整理 baseline writer 的临时输出，生成官方提交目录结构：

```bash
python tools/local_collect_predictions.py \
  --raw-dir /root/autodl-tmp/jittor_pcd/submit_result_raw \
  --out-dir /root/autodl-tmp/jittor_pcd/submit_result \
  --list datalist/test.txt \
  --input-prefix dataset_test_noisy \
  --filename denoised.npy \
  --overwrite
```

检查数量和点数：

```bash
find /root/autodl-tmp/jittor_pcd/submit_result -name "denoised.npy" | wc -l
find dataset_test_noisy -name "noisy.npy" | wc -l
```

打包成 `result.zip`：

```bash
cd /root/autodl-tmp/jittor_pcd/submit_result
zip -r /root/autodl-tmp/jittor_pcd/result.zip shapenet
```

最终提交文件是：

```text
/root/autodl-tmp/jittor_pcd/result.zip
```

压缩包内部应该长这样：

```text
shapenet/<synset_id>/<model_id>/denoised.npy
```
